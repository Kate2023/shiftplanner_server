The API between the frontend and backend

[//]: # (User Authentication API)

API 1. User Authentication
Endpoint: POST /api/auth/login
Description: Authenticates a user and returns a JWT token for subsequent requests.
Request Body:
{
"role": "string",
"username": "string",
"password": "string"
}
Response Code:
{
"200 OK": "Authentication successful",
"401 Unauthorized": "Invalid credentials"
}
Response body on success:
{
"token": "string",
"userId": "integer",
"role": "string"
}

API 9. create schedule 
Endpoint: POST /api/schedules/{date}
Description:
    Following the below steps to do the Auto-scheduling:
        1.If a given schedule is again the policies, it will be return 400 with reason.
        2.Otherwise, it will try to replace Optional with the policies.
        3.If found a solution, saved to the database and return 201. 
        4.If no solution, it will return 400 with reason.

Request headers:
{
"token": "string"
}
Request Body:
{
"date": "string",
"rosterStaffId": "integer",
"bankingStaffId": "integer",
"backupStaffId": "integer",
"inspectionStaffId": "integer",
"notes": "string",
"policies": [
"policyId": "integer"
],
"assignments": [    
{
"staffId": "integer",
"timeSlot": "string",
"taskId": "integer"
},
...
]
}
Response Code:
{
"201 Created": "Saved successfully",
"400 Bad Request": "Invalid input data",
"401 Unauthorized": "Invalid credentials",
"404 Not Found": "Schedule not found for the specified date"
}
Response Body on success:
{}
Response Body on failure:
{
"reason": "string"
}
