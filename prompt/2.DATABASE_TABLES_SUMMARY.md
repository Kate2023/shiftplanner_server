# Shift Planner - Database Tables Summary

## Overview
This document summarizes all the data tables required for the Shift Planner application for Auckland Library, supporting Manager and Librarian user roles.

---

## Core Database Tables

### 1. **Users Table**
Stores authentication information for both Manager and Librarian roles.
| Column | Type | Constraints | Description |
|--------|------|-----------|------------|
| user_id | INT | PRIMARY KEY, AUTO_INCREMENT | Unique user identifier |
| username | VARCHAR(255) | UNIQUE, NOT NULL | Username for login |
| password | VARCHAR(255) | NOT NULL | Encrypted password |
| role | ENUM('Manager', 'Librarian') | NOT NULL | User role |

---

### 2. **Staff Table**
Stores staff member information used in the roster.
| Column | Type | Constraints | Description |
|--------|------|-----------|------------|
| staff_id | INT | PRIMARY KEY, AUTO_INCREMENT | Unique staff identifier |
| staff_name | VARCHAR(255) | NOT NULL | Name of the staff member |
| working_hours_per_week | DECIMAL(5,2) | NOT NULL | Total working hours per week |
| lunch_break | DECIMAL(5,2) | NOT NULL | Lunch break duration in minutes |
| is_active | BOOLEAN | DEFAULT TRUE | Active status |

---

### 3. **Tasks Table**
Defines all operational and non-operational tasks that can be assigned to staff.
| Column | Type | Constraints | Description |
|--------|------|-----------|------------|
| task_id | INT | PRIMARY KEY, AUTO_INCREMENT | Unique task identifier |
| task_name | VARCHAR(255) | UNIQUE, NOT NULL | Name of the task (e.g., Desk, Check-in, Roaming, etc.) |
| colour | VARCHAR(7) | NOT NULL | Hex colour code for UI display |
| is_lunch | BOOLEAN | DEFAULT FALSE | Marks lunch-related tasks used between 12:00-14:00 |
| is_auto | BOOLEAN | DEFAULT FALSE | Marks tasks that can be auto-assigned when filling optional slots |

**Task Types to include:**
- Operational: Desk, Roaming, Check-in, Picking, Shelving
- Break: Lunch, Lunch/Check-in, Lunch/Bell, Lunch/Roaming
- Admin: Bell, Meeting, Closing, Event Prep, Daily Building Inspection
- Event: Event
- Optional: Unassigned (to be replaced with actual tasks during scheduling)

### 4. **Schedules Table** (Roster/Timetable)
Main schedule/roster for a specific date.
| Column | Type | Constraints | Description |
|--------|------|-----------|------------|
| schedule_id | INT | PRIMARY KEY, AUTO_INCREMENT | Unique schedule identifier |
| date | DATE | NOT NULL | The date of this schedule |
| notes | TEXT | NULL | Additional notes for the schedule |
| roster_staff_id | INT | FOREIGN KEY (staff.staff_id) | Staff member responsible for creating the roster |
| banking_staff_id | INT | FOREIGN KEY (staff.staff_id) | Staff member responsible for banking |
| banking_backup_staff_id | INT | FOREIGN KEY (staff.staff_id) | Backup staff for banking |
| daily_building_inspection_staff_id | INT | FOREIGN KEY (staff.staff_id) | Staff member assigned to daily building inspection |
---

### 5. **Schedule_Assignments Table** (Time Slot Assignments)
Individual task assignments for each staff member in a time slot.
| Column | Type | Constraints | Description |
|--------|------|-----------|------------|
| assignment_id | INT | PRIMARY KEY, AUTO_INCREMENT | Unique assignment identifier |
| schedule_id | INT | FOREIGN KEY (schedules.schedule_id) | Reference to schedule |
| staff_id | INT | FOREIGN KEY (staff.staff_id) | Assigned staff member |
| time_slot_start | TIME | NOT NULL | Start time of the slot (e.g., 08:00) |
| time_slot_end | TIME | NOT NULL | End time of the slot (e.g., 09:00) |
| task_id | INT | FOREIGN KEY (tasks.task_id) | Assigned task |

---

## Summary of Tables

| Table Name | Purpose | Key Relationships |
|-----------|---------|-------------------|
| **Users** | Authentication & User Management | - |
| **Staff** | Staff Member Information | Referenced by Schedules and Schedule_Assignments |
| **Tasks** | Task Definitions & Colours | Referenced by Schedule_Assignments |
| **Schedules** | Daily Roster/Timetable | Parent table for Schedule_Assignments |
| **Schedule_Assignments** | Individual Time Slot Assignments | Joins Schedules, Staff, and Tasks |

## Entity Relations Diagram (Textual)

```
Users
  ├── creates → Schedules
  
Schedules
  ├── has many → Schedule_Assignments
  └── references → Staff (roster, banking, backup, inspection)
  
Staff
  └── assigned in → Schedules and Schedule_Assignments
  
Tasks
  └── used in → Schedule_Assignments
```

---

## Key Design Considerations

1. **Time Slots**: Represented as TIME fields with start and end times
2. **Colours for Tasks**: Stored as hex codes (e.g., #FF5733) for UI rendering
3. **Role-Based Access**: Users table distinguishes between Manager and Librarian
4. **Daily Role Fields**: Roster, banking, backup, and inspection are stored directly on each schedule record
5. **Optional Slots**: Optional coverage is determined by task configuration and assignment logic
6. **Staff Active Status**: Allows soft-deletion and deactivation without data loss
7. **Policy Rules**: Scheduling policies are enforced in application logic rather than separate policy tables
